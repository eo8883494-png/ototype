# TikTok Content Posting API 申請キット(2026-07-18準備)

目的: ブラウザ経由の投稿自動化を廃止し、公式APIによる完全無人投稿へ移行する。
審査通過までは現行のTelegram半自動運用を継続。

## あなた(人間)がやる手順

アカウント登録・ログインが必要なので、以下はユーザー側の操作です(所要15〜30分)。
迷ったらこのファイルを見ながらでも、僕に画面を見せながらでもOK。

1. https://developers.tiktok.com/ を開き、TikTokアカウント(@運用アカウント)でログイン
2. 右上「Manage apps」→「Connect an app」で新規アプリ作成
3. 各欄は下の「コピペ用申請内容」をそのまま使う
4. Products から **Content Posting API** を追加し、スコープ `video.publish` をリクエスト
5. 審査(App review)に進み、下の Use case 文を貼る。デモ動画/スクショを求められたら僕に言ってください(その時点でscripts/tiktok_api_post.pyを動かして画面収録の段取りをします)
6. 申請後は待ち(目安3日〜2週間、差し戻しがあれば都度対応)

## コピペ用申請内容

- **App name**: Ototype Video Poster
- **Category**: Entertainment
- **Description (EN)**:
  A private, single-operator tool that posts short personality-quiz result videos
  (music-listening type diagnosis "Ototype") to the operator's own TikTok account
  on a daily schedule. The videos are generated by the operator's own rendering
  pipeline. The app is not distributed to any third party and handles no
  third-party user data.
- **Privacy policy URL**: https://eo8883494-png.github.io/ototype/tiktok-poster-privacy.html
- **Scopes**: video.publish (Direct Post)
- **Use case (EN)**:
  We operate "Ototype" (https://eo8883494-png.github.io/ototype/), a free
  music-listening personality quiz. Our pipeline renders 8-second vertical
  videos introducing each personality type. This app uploads 1-3 of those
  videos per day to our own TikTok account (no other accounts). We need the
  video.publish scope to replace a manual upload workflow. The app is
  operator-internal: a single OAuth authorization for our own account,
  no third-party users, no data collection.

## 審査前にできること/できないこと(仕様メモ)

- 審査前はSELF_ONLY(自分のみ公開)でのテスト投稿が可能 → デモ収録はこれで撮る
- APIには予約投稿パラメータが無い → 投稿時刻の制御は現行のscheduled task側で行う(19:00実行のまま)
- トークンは60〜90日で要リフレッシュ → tiktok_api_post.pyにリフレッシュ処理を実装済みにしておく

## 承認後の切替手順(僕がやる)

1. OAuth初回認可(ブラウザで1回、あなたの承認ボタンだけ必要)
2. トークンを x-autopilot/.env に保存
3. ototype-tiktok-daily の手順5を「send_to_phone.py」→「tiktok_api_post.py」に差し替え
4. 1本目はSELF_ONLYでテスト→問題なければ公開設定で運用開始、Telegram通知は「投稿完了報告」に役割変更
